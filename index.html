Why this dataset is valuable and messy

Debt markets move on details that rarely arrive in clean tables.

New term loans, revolver amendments, covenant waivers, defaults, maturity pushes, refinancings, and guarantor changes can materially alter credit risk. The problem is that the underlying facts are scattered across SEC 8-K items, exhibit indexes, amended credit agreements, indentures, waiver letters, and other legal attachments.

High-value workflow

Credit investors, diligence teams, and monitoring systems care about these events quickly because they can change risk, pricing, and escalation paths.

Source fragmentation

Relevant terms may appear in Item 1.01, 2.03, 2.04, or 8.01, then continue inside exhibits where definitions and amendments live.

Operational pain

Analysts still read long legal prose manually. Agents need explicit fields, source links, and confidence metadata rather than loose summaries.

If the choice is between overstating certainty and showing the source, show the source.

What DebtSignal is

A source-linked event feed for SEC financing disclosures, designed to be useful before full legal parsing is solved.

DebtSignal is a docs-first API concept for detecting financing and covenant events in SEC 8-K filings and related exhibits. Each event is intended to return issuer identity, filing metadata, event type, normalized terms where extractable, source excerpts, filing links, and explicit confidence and completeness signals.

Who it is for

  • Credit and event-driven funds monitoring new debt, amendments, waivers, defaults, and maturity changes.
  • Diligence and private credit teams reviewing borrower disclosure history with direct filing provenance.
  • Fintech and AI-agent builders triggering workflows from debt-related disclosure events.
  • Research and data engineering teams building issuer timelines and joining event data with broader datasets.

Honest v1 scope

  • Docs-first conceptual API and schema.
  • Source-linked event objects and machine-readable docs.
  • Designed for monitoring and triage, not exhaustive live coverage.
  • Not a legal opinion engine, full covenant compliance engine, or substitute for reading the source documents.

Agent-friendly design principles

Compact surface area, stable shapes, explicit uncertainty.

Stable identifiers

Every record should expose a durable event_id plus filing accession data to support joins, caching, and replayable workflows.

Provenance first

Each extraction should include filing URLs, exhibit references, and excerpts so humans and agents can verify claims without guessing.

Nulls over fiction

Fields that are not confidently extractable should remain null, paired with confidence and completeness metadata instead of synthetic certainty.

Minimal endpoint set

Small enough for agents to integrate quickly, clear enough for buyers to evaluate honestly.

Machine-readable docs are available at docs/openapi.json, with the conceptual docs endpoint surfaced as GET /v1/docs/openapi.json.

Launch surface note: the docs-first entrypoint includes a machine-readable spec endpoint so agents and evaluators can discover the schema without guessing.

GET

/v1/docs/openapi.json

Return the conceptual OpenAPI 3.1 document for the v1 surface so buyers, developers, and agents can inspect the schema directly.

GET

/v1/events

List financing and covenant-related events extracted from SEC 8-K filings with filters like issuer, ticker, event type, filing date, accession number, exhibit presence, confidence, and cursor pagination.

GET

/v1/events/{event_id}

Fetch a single event with richer extraction payloads, source excerpts, exhibit references, confidence signals, and extraction notes.

GET

/v1/issuers/{issuer_id}/events

Return an issuer-specific financing disclosure timeline for monitoring, diligence, and borrower review use cases.

GET

/v1/event-types

Expose the initial classification vocabulary: new financing, amendment, waiver, default or acceleration, covenant change, maturity extension, refinancing, repayment or termination, collateral or guarantor change, and liquidity update.

Roadmap, not live today

x402-ready future positioning

DebtSignal is positioned to support usage-based or agent-mediated payment flows later without changing the core event model. That is roadmap positioning only. Billing, access control, and payment flows are not live in this docs-first launch.

Caveats and limitations

Clear boundaries matter more than inflated claims.

  • Coverage is limited to publicly disclosed financing and covenant events in SEC filings and available exhibits.
  • Disclosure quality varies by issuer, filing timing, terminology, and redaction practices.
  • Some extracted terms will be partial or null because the source language is ambiguous or spread across documents.
  • High-stakes workflows still require human review of the original filing and exhibit text.
  • The launch posture is docs-first and conceptual, not a claim of exhaustive live extraction coverage across all issuers.
  • DebtSignal is not a legal parser, covenant compliance engine, or substitute for counsel.
