DebtSignal is live at https://debtsignal-site.onrender.com

DebtSignal is a docs-first API concept for financing and covenant events disclosed in SEC 8-K filings and related exhibits. The aim is to turn messy public debt disclosures into source-linked, agent-friendly event data.

Why this dataset is useful:
Debt raises, amendments, waivers, defaults, covenant changes, maturity extensions, refinancings, and related events are meaningful signals for credit, diligence, and automated research workflows. The data is messy because the signal is scattered across 8-K text and exhibits, issuer disclosure quality and timing vary, and key terms can be partial, ambiguous, redacted, or null.

What is live now:
- Site: https://debtsignal-site.onrender.com
- Repo: https://github.com/xoaxza/debtsignal-site
- Machine-readable docs artifacts: https://debtsignal-site.onrender.com/docs/openapi.json and https://debtsignal-site.onrender.com/docs/sample-event.json
- Conceptual v1 endpoint surface documented on the site:
  - GET /v1/events
  - GET /v1/events/{event_id}
  - GET /v1/issuers/{issuer_id}/events
  - GET /v1/event-types
  - GET /v1/docs/openapi.json

Positioning:
- Source-linked monitoring and triage layer built on public SEC disclosures
- Response model emphasizes provenance, confidence, completeness flags, and explicit nulls
- x402-ready in the future: the schema is compatible with adding usage-based or agent-mediated payments later, but billing and access control are not live today

Main caveats:
- This is a docs-first concept site, not a claim of exhaustive live extraction coverage or a production extraction API
- Coverage is limited to publicly disclosed events in SEC filings and available exhibits
- DebtSignal is not a legal opinion, covenant compliance engine, or substitute for reading the source documents
- Human review is still appropriate for legal, underwriting, or high-stakes investment decisions
